# RFP-AutoPilot (标书智能应答与格式化工具)

> **产品定位**：一款专为 B2B 招投标场景打造的“效率神器”，解决技术标书制作中痛点最痛、耗时最长的条款应答与格式调整问题。

## 💡 痛点与背景 (Background & Pain Points)

在 B2B 业务的招投标过程中，售前技术支持和标书制作人员常常面临以下挑战：

1.  **条款繁多，应答耗时**：一份技术规范书往往包含数百条技术条款，逐条阅读并进行“满足”或“优于”的应答，耗费大量人力时间。
2.  **内容重复，复用困难**：针对同一类产品，不同项目的技术要求大同小异，历史标书中的优质应答难以被高效复用，往往只能低效地“复制粘贴”。
3.  **格式杂乱，调整费力**：甲方提供的 Word 文档格式千奇百怪（自动编号混乱、字体不一、表格错位），人工调整格式不仅枯燥，而且容易出错。
4.  **市场空白，自主摸索**：市面上针对 Word 文档“智能应答 + 深度格式化”的垂直领域工具几乎为空白，通用文档处理软件无法满足招投标的特定业务逻辑。

基于以上痛点，本项目**从 0 到 1 自主摸索**，设计并实现了一套完整的自动化解决方案，显著提升标书制作效率与质量。

## ✨ 核心功能 (Core Features)

### 1. 🤖 智能应答 (Smart Response)
系统如同一个经验丰富的“投标助手”，自动理解标书并生成应答：
- **精准条款识别**：基于复杂的正则算法，精准提取文档中的各类编号条款（支持 `1.1`、`（一）`、`①` 等多种复杂层级格式）。
- **双模应答策略**：
    - **标准应答**：快速生成“我方完全满足招标文件要求”的通用应答。
    - **智能应答**：基于**知识库（产品模板）**，利用全文本相似度算法（SequenceMatcher），自动匹配历史最优应答。
- **动态知识库**：用户可根据不同产品（如“产品A”、“产品B”）维护不同的 `.docx` 模板，系统自动同步，实现经验沉淀。
- **智能组合匹配**：独创“父子条款组合匹配”逻辑，有效解决短条款（如仅有“1.7”）因上下文缺失导致的误匹配问题。

### 2. 📝 深度格式化 (Deep Formatting)
一键解决“格式强迫症”，输出符合专业标书规范的文档：
- **自动编号“冻结”**：将 Word 自动编号转换为静态文本，彻底解决不同电脑打开编号错乱的顽疾。
- **样式标准化**：统一全文档字体（中文宋体/英文 Times New Roman）、行距、缩进，自动清理多余空行。
- **表格与图片优化**：自动美化表格边框与字体，自动居中并缩放超宽图片，确保版面整洁。

### 3. 🔍 辅助检查 (Assisted Check)
- **关键词高亮**：自动标青“提供”、“出具”、“检验报告”等关键动作词，辅助人工复核，防止漏项。
- **关键符号标红**：自动识别并标红 `*`、`★`、`※` 等核心参数标记，确保关键指标不被遗漏。

## 📖 使用流程 (User Workflow)

产品的设计遵循“简单、直观、可控”的原则：

### Step 1: 准备工作
- 整理你现有的高质量标书应答，保存为 `.docx` 文件（如 `领域1-产品A.docx`）。
- 将这些文件放入软件根目录下的 `回应条款` 文件夹中。
- **亮点**：系统会自动扫描该文件夹，将文件名作为“产品列表”同步到软件界面，无需修改代码即可扩展产品库。

### Step 2: 导入标书
- 打开软件，点击“选择文件”按钮。
- 选择甲方提供的招标技术规范书（支持 `.doc` 和 `.docx` 格式）。

### Step 3: 选择模式
根据实际需求选择处理模式：
- **格式调整**：仅对文档进行美化和编号处理。
- **智能应答**：
    - **常规模式**：不选择具体产品，系统将生成标准通用应答。
    - **智能模式**：在下拉菜单中选择一个产品（如 `领域1-产品A.docx`）。系统将加载对应的知识库，进行相似度匹配。

### Step 4: 一键处理
- 点击“开始处理”，软件将自动运行。
- 处理完成后，会在原文件同级目录下生成 `-格式调整.docx`或`-智能应答.docx` 文件。

## 🚀 快速开始 (Quick Start)

### 环境要求
- Windows 10/11
- Python 3.8+

### 安装与运行

1. **克隆项目**
   ```bash
   git clone https://github.com/tuke-workspace/RFP-AutoPilot.git
   cd word-smart-processor
   ```

2. **安装依赖**
   ```bash
   pip install -r requirements.txt
   ```

3. **启动应用**
   ```bash
   python src/main_word_processor.py
   ```

### 打包发布
生成无需 Python 环境的 `.exe` 可执行文件，方便分发给同事使用：
```bash
cd src
quick_build.bat
```

## �️ 技术架构 (Architecture)

项目采用模块化分层设计，确保了代码的可维护性与扩展性：

```mermaid
graph TD
    UI[用户界面 (PyQt5)] --> Controller[业务编排 (DocumentProcessor)]
    
    subgraph Core[核心引擎]
        Controller --> Format[格式化模块]
        Controller --> Response[智能应答模块]
    end
    
    subgraph Algorithms[算法层]
        Response --> Clause[条款层级分析 (Regex)]
        Response --> Similarity[文本相似度匹配 (Difflib)]
        Response --> Knowledge[动态知识库加载]
    end
    
    Format --> PythonDocx[python-docx]
    Knowledge --> FileSystem[文件系统]
```
